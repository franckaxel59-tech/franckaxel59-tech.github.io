       1.1 C ONTEXTE        GÉNÉRAL



       Suite à son expansion à Toulouse, le studio de création PixelForge a décidé de
basculer l'intégralité de son infrastructure sous des technologies Open-Source.
L'objectif est double : s'affranchir des licences propriétaires et automatiser la gestion
du parc informatique.
Votre équipe décide de concevoir un environnement de production 100% Linux
(Debian/Ubuntu) intégrant un annuaire centralisé, des services cloud conteneurisés et
un système de déploiement automatisé.
Contraintes techniques majeures :
   ·   Système : Environnement exclusivement Linux (Serveurs et Postes clients).
   ·   Identité : Centralisation des comptes via un contrôleur de domaine Samba 4
       AD.
   ·   Services : Plateforme de stockage Nextcloud déployée sous Docker ou
       equivalent.
   ·   Déploiement : Configuration des postes clients via Ansible (zéro configuration
       manuelle).
   ·   Gestion des accès : Seuls les administrateurs du domaine AD doivent avoir des
       droits sudo sur les machines Linux.

5 É VALUATIONS CROISÉES
Pour valider le projet, chaque groupe devra passer les tests suivants :
   1 Création d'un user "Admin_Toulouse" dans l'AD : il doit être sudo sur tous les
     postes Linux.
   2 Création d'un user "Collaborateur" : il ne doit pas être sudo mais doit pouvoir se
     connecter à Nextcloud.
   3 Redémarrage complet : Tout l'environnement (Conteneurs, AD, Partages) doit
     remonter automatiquement au boot.
